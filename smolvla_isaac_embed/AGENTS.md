# AGENTS.md

本文件用于指导 AI 助手在 `smolvla_isaac_embed/` 中协作开发。

## 项目定位

这是一个围绕 `SmolVLA + Isaac Sim / IsaacLab` 的集成工程，不是 `lerobot` 上游源码本体。

本工程的主要职责是：

- 管理运行脚本。
- 管理配置文件。
- 编写环境适配层。
- 编写动作适配层。
- 编写安全测试包装层。
- 记录实验与失败案例。

AI 助手应默认把 `lerobot/` 视为上游依赖，把本工程视为“薄集成层”。

## 开发边界

优先在本工程中实现：

- 观测结构检查工具
- 命名映射
- 动作重排
- 安全测试包装层
- 实验脚本与日志组织

只有在以下情况才应修改 `lerobot/`：

- 上游已有明确 bug，阻塞当前流程。
- 必须增加一个极小的接口暴露点，且无法在外层适配。
- 需要验证某个上游实现细节。

若修改 `lerobot/`，应优先：

- 最小改动
- 明确记录原因
- 在文档里说明修改点

## 文档语言

除非用户明确要求英文，否则 Markdown 文档默认使用中文。

代码注释可根据上下文决定中文或英文，但以简洁准确为主。

## 推荐工作方式

AI 助手在处理任务时，应优先：

1. 先识别当前任务属于哪一层：
   - 配置层
   - 脚本层
   - 观测适配层
   - 动作适配层
   - 安全包装层
   - 实验记录层
2. 优先修改本工程目录，而不是直接改上游。
3. 若任务涉及模型输入输出，优先检查：
   - `state_keys`
   - `camera_keys`
   - `rename_map`
   - `action_dim`
   - 动作顺序
4. 若任务涉及性能或显存，优先检查：
   - `batch_size`
   - 相机数量与分辨率
   - `headless`
   - 视频录制开关

## 当前学习阶段建议

当前阶段优先目标：

1. 跑通现成 `SmolVLA` + Isaac 环境闭环。
2. 写清楚输入输出接口。
3. 加入一个最小安全测试模块。

当前阶段暂不优先：

- 通用平台化
- ROS2 全面接入
- 多模型统一服务层
- 复杂前端或可视化系统

## 文档沉淀要求

以下信息不应只留在聊天记录中，而应写入文档：

- 已验证可运行的命令
- 环境版本
- 显存占用结论
- 失败原因与解决方式
- 模型与环境接口映射

建议 AI 助手在完成关键阶段后，主动补充：

- `experiments/notes.md`
- `experiments/failures.md`
- `README.md`
